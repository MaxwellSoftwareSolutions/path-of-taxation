# Vertical Slice

## Scope

The vertical slice exists to answer one question:

Can this game deliver a run that feels good enough to replay immediately?

If the answer is unclear, adding content is waste.

## Included

- One hub screen
- One playable character
- One biome
- One boss
- Five regular enemy types
- Two elite enemy variants
- Two active skills plus dash and panic skill
- 20 run modifiers
- 15 loot items
- One turn-based minigame
- One meta progression track with sidegrade unlocks

## Excluded

- Additional classes
- Additional biomes
- Online features
- Crafting economy
- Trading systems
- Large passive web
- Full story mode
- More than one minigame

## Success Criteria

- A run is playable from start to finish in 10 to 20 minutes.
- Combat feels responsive and readable.
- At least 3 noticeably different builds can emerge from the same starting kit.
- The minigame improves pacing instead of stalling it.
- Dying still leaves the player with a meaningful unlock or new option.

## Risks

### Risk: combat is shallow

Mitigation:

- Prototype enemy behavior before building content quantity.
- Focus early on hit feel, mobility, and telegraphs.

### Risk: satire overwhelms readability

Mitigation:

- Separate flavor text from mechanical text.
- Put gameplay clarity above jokes in every interface.

### Risk: minigame feels disconnected

Mitigation:

- Make the reward affect the next run immediately.
- Keep sessions short and deterministic.

### Risk: build choices are fake

Mitigation:

- Ensure weird choices can become strong with support.
- Avoid obviously dominant generic upgrades.

## Development Order

1. Camera, movement, and room transitions
2. One complete player kit
3. One complete enemy archetype
4. Reward selection flow
5. Boss encounter
6. Hub return flow
7. Turn-based minigame
8. Meta progression
9. Additional enemies and modifiers
