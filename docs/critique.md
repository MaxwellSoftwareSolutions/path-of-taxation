# Plan Critique

## What This Project Is At Risk Of Becoming

The most likely failure mode is not technical. It is creative overreach.

There is a strong temptation to build:

- a giant passive tree
- huge loot pools
- many classes
- multiple satire systems
- fake endgame layers

That would produce a broad but weak prototype.

## The Good Constraint

The game should be judged on one run, not on the promise of future scale.

If one run is not exciting, no amount of added systems will save it.

## Hard Criticism Of The Current Direction

### The idea can become too referential

If every joke depends on knowing a specific ARPG scene reference, the project narrows its audience and dates itself fast.

Correction:

- make the jokes legible to any player who knows optimization culture
- use ARPG specificity as texture, not as the only source of humor

### The action side could be underfunded

It is easy to spend design energy on item jokes and fake complexity while leaving the actual combat thin.

Correction:

- the first implementation milestone should be movement, enemy pressure, and hit feel
- no second system should be added until the first combat room is fun

### The turn-based minigame could feel mandatory instead of rewarding

If the minigame becomes homework between runs, it will damage retention.

Correction:

- keep it short
- give immediate strategic rewards
- allow the player to understand it in seconds

### Scope pressure will appear early

Satire invites idea generation, which is dangerous during pre-production.

Correction:

- add new content only after a fun vertical slice
- reject any feature that does not strengthen the main loop directly

### The parody can drift too close to source material

Being too recognizable creates legal and creative weakness.

Correction:

- parody structures, player behavior, and design philosophy
- avoid copying names, lore, iconography, or exact mechanics

## Standard For Expansion

Do not add a feature unless it satisfies at least one of these:

- improves combat feel
- increases build diversity
- improves run pacing
- creates replay motivation

If it only adds flavor, it is optional.
